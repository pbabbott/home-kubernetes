Source: `[infrastructure/controllers/ingress-nginx/ingress-nginx-helm-release.yaml](../../../infrastructure/controllers/ingress-nginx/ingress-nginx-helm-release.yaml)`  
Istio install: `[istio-ingress-helm-release.yaml](istio-ingress-helm-release.yaml)` and related HelmReleases in this directory.

This is a **best-effort** translation. Items without a 1:1 Helm value are called out below.

## Operational notes

- `**numTrustedProxies**`: Set to the number of trusted reverse proxies **in front of the gateway** that append to `X-Forwarded-For`. If you only terminate TCP with PROXY protocol and have no extra HTTP proxies, `1` is a reasonable starting point; **validate** against your actual topology.
- **PROXY protocol + TLS**: Ensure your L4 load balancer sends PROXY headers on the same listener where TLS is terminated or passed through, matching how nginx saw traffic.
- **Ambient + gateway**: Your mesh uses **ambient** profile (`istiod` / CNI / ztunnel). The ingress `**gateway`** chart still provides a classic dataplane ingress; keep `dependsOn` ordering as-is so ztunnel/CNI are ready before the gateway rolls out.
